Fix globe rendering tiny/uncentered: set explicit initial camera view

Page was loading correctly (the crash is fixed), but the globe itself
rendered as a small, off-position circle in the corner instead of
filling the view. Cause: Cesium's raw default camera position is an
awkward tight zoom, not a clean full-Earth framing, and we never set
one explicitly. Combined with requestRenderMode (only redraws on
scene changes), it just sat at whatever uninitialized resting state
it started in.

Fix: explicitly set the camera to a top-down view ~25,000km out on
mount (roughly 4x Earth's radius, framing the whole globe with
margin), then call scene.requestRender() once since requestRenderMode
needs an explicit trigger for changes Cesium's own input handlers
didn't cause.

User-driven camera movement (drag/zoom) was already fine — Cesium's
camera controller flags those as scene changes itself under
requestRenderMode; only our React-driven updates and this initial
view needed manual triggering.
